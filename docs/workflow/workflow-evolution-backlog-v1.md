# 运行进化待办 v1

## 用途

把“已经意识到值得优化，但还没落地”的事项集中管理，避免反复临时思考。

## 记录格式

| 优先级 | 事项 | 目标 | 触发条件 | 预期收益 | 状态 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| P0 | 建立图片替换一致性更强的 smoke 保护 | 防止真图替换后串图、空图、状态错配 | 每次替换真实图片 | 直接降低图片回归风险 | 进行中 |
| P0 | 浏览器 E2E 在环境可用后接回 | 把静态通过升级为真实交互通过 | 本地端口环境放开 | 补齐最终 QA 闭环 | 待触发 |
| P1 | 为角色日志建立统一的接任记录模板 | 让接任动作更标准 | 出现新接任时 | 缩短小弟重新上手时间 | 待做 |
| P1 | 为高优先级钓点建立图片来源采集脚本 | 减少手工追图成本 | 真图替换进入密集期 | 节省算力、减少重复劳动 | 待做 |
| P1 | 给 `spot-enrichment.json` 加结构校验脚本 | 防止字段补强时破坏形状 | 字段继续扩展时 | 降低数据层错误 | 待做 |
| P2 | 真图替换后增加截图对比 QA | 让视觉回归更直观 | 浏览器回归可用后 | 提高 UI 稳定性 | 待触发 |

## 使用规则

- 只放“明确值得做但这轮不一定立刻做”的优化
- 做完就移除或改成“已完成”
- 如果某项连续两轮都阻碍主线，应提升优先级
