# 运行错误与防错账本 v1

## 用途

记录执行过程里的真实错误，避免以后在同类问题上重复浪费算力。

## 记录格式

| 日期 | 错误 | 原因 | 修正动作 | 以后默认做法 | 状态 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 2026-04-14 | 小弟线程失效后直接新开接任者 | 缺少明确的复用优先规则 | 建立角色复用与交接协议 | 同角色优先复用原小弟，不可用再接任 | 已固化 |
| 2026-04-14 | 早期只有总协作档案，没有角色专属职责和日志 | 交接依赖临时上下文 | 补建每个角色的职责文档和日志文件 | 每个角色都要有 `role + log` 双文件 | 已固化 |
| 2026-04-14 | 临时用 Node 检查补强数据时误把对象当数组 | 对 JSON 结构假设不严谨 | 改成按对象键访问并补检查脚本 | 对数据结构先检查 `Object.keys` / 形状，再取值 | 已固化 |
| 2026-04-14 | 尝试在当前环境直接跑本地端口浏览器回归 | 环境限制未充分前置 | 先用静态 smoke 顶住，再保留浏览器脚本 | 端口受限时优先静态回归，不阻塞主线 | 已固化 |
| 2026-04-15 | 尝试用 Playwright MCP 取外网页面图时遭遇环境错误 | 当前环境对浏览器目录和页面加载有限制 | 回到静态与远程资源替换主线 | 当前环境下优先用数据层和可验证公开资源推进 | 已固化 |
| 2026-04-15 | 图片替换阶段存在“图换了、状态没换”的风险 | 图片字段和前台状态文案是两套信息 | 补图片状态一致性断言 | 真图替换后先跑 smoke，再继续下一张图 | 已固化 |
| 2026-04-15 | 在 `build.mjs` 模板字符串里直接写反引号导致构建失败 | 文案修改时忽略了模板字符串语法边界 | 去掉模板里的反引号并立即重跑 smoke | 改模板文案时避免直接插入反引号，改后立即先跑构建 | 已固化 |
| 2026-04-15 | 用一条串联 shell 命令做目录创建+打包被策略拦截 | 当前环境对命令形态更保守，且串联命令不利于稳定执行 | 改成分步单命令执行 | 需要文件操作时优先拆成单步命令，不用 `&&` 串联 | 已固化 |

## 使用规则

- 只记录真实发生过的错误
- 每条错误都必须写出以后默认做法
- 如果同类错误再次出现，必须把默认做法升级成更硬的机制
