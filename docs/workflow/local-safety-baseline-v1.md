# 本地安全底线 v1

## 终极底层标准

任何操作都不得要求关闭、绕过或削弱你本机已有的系统、浏览器、账号、文件权限或网络安全边界；凡是需要 `--no-sandbox`、关闭系统安全机制、提升到 `sudo`、明文落地密钥、粗暴放宽权限、执行未审查远程脚本的做法，一律视为不合格方案，禁止作为交付路径。

## 强制执行规则

1. 不用 `--no-sandbox`、`--disable-setuid-sandbox`
2. 不把本地预览、测试、部署建立在 `sudo` 上
3. 不把 GitHub Token、SSH 私钥、账号密码写进仓库、脚本或日志
4. 不要求用户关闭系统防火墙、Gatekeeper、SIP、浏览器安全策略
5. 不使用 `curl ... | sh` 这一类未审查远程执行链
6. 不用 `chmod -R 777` 这类粗暴权限修改
7. 不执行破坏性系统命令，不越出项目目录乱写文件

## 当前项目的执行落点

- 本地预览前，先跑：
  - `npm run safety:preflight`
- `site/preview-local.command` 已接入这道预检
- 阶段性交付前，除了大南 QA Gate，还要通过本地安全底线

## 失败处理

只要命中上面的禁止项：

- 立即停止该路径
- 不继续“先跑起来再说”
- 改走不削弱本地安全边界的替代方案
