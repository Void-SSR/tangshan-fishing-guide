# 唐山钓鱼指南 UX 架构 v2

## 核心指导思想

这个网站虽然是网页实现，但产品体验按手机 App 来做。

核心不是做一个资讯门户，而是做一个围绕钓点展开的决策工具。所有页面都要围绕一个目标服务：

- 优先推荐鱼更多、表现更强、稳定性更高的钓点

其他信息都是辅助用户决策的支持信息。

## 三层结构总原则

全站使用严格的三层跳转结构：

1. 第一层级：模式选择
2. 第二层级：钓点列表
3. 第三层级：单钓点详情

最重要的交互约束：

- 从第一层级进入第二层级后，不再出现第一层级的大按钮 UI
- 从第二层级进入第三层级后，不再保留第二层级的大列表 UI
- 第三层级是单独的详情页，不是列表页内展开

## 第一层级：首页

首页只做两种模式选择：

- 海岸钓
- 出海钓

首页不要承载复杂筛选，不要把大量钓点信息直接堆在首页。

首页的任务只有三个：

1. 让用户明确进入哪种模式
2. 告诉用户这是一个“按钓点排名和实战价值”组织的指南
3. 提供一个明确、快速、可信的入口

### 首页建议模块

#### 1. 顶部品牌区

- 产品名：唐山钓鱼指南
- 一句话说明：围绕海岸钓与出海钓，优先推荐更强、更稳、更值得去的钓点

#### 2. 模式入口区

只放两个大入口卡：

- 海岸钓
- 出海钓

每个入口卡建议带：

- 封面图
- 模式说明
- 一句辅助文案

#### 3. 底部辅助信息

只保留轻量内容：

- 最新更新时间
- 合规提醒入口
- 关于排名说明入口

首页不建议出现：

- 第一屏就出现大量钓点列表
- 第一屏就出现复杂筛选器
- 船钓和岸钓混在一起展示

## 第二层级：钓点基本信息列表页

以海岸钓为例，点击首页的“海岸钓”后，进入海岸钓列表页。

出海钓同理。

这个页面只展示当前模式下的所有钓点，不再出现首页的“海岸钓 / 出海钓”双入口大按钮。

### 第二层级页面目标

它是一个钓点浏览与比较页，核心任务是：

- 让用户快速浏览这个模式下最值得去的钓点
- 让用户按排名顺序看点
- 让用户快速进入单个钓点详情

### 第二层级排序逻辑

默认按钓点强度排序，也就是按你资料里的推荐优先级排序：

- 第一梯队在最上
- 第二梯队在中间
- 第三梯队在下方

同梯队内按实测表现、核心目标鱼表现、稳定性和友好度排序。

### 第二层级每张钓点卡必须包含的内容

- 钓点图片
- 钓点名称
- 标签：海岸钓 / 出海钓
- 区域与导航信息
- 核心钓段说明
- 主打鱼种
- 核心汛期
- 排名标识
- “展开详细”按钮

### 第二层级推荐补充信息

可以补但不要压过核心内容：

- 新手友好度简标
- 今日适合度占位
- 风险简标
- 停车或登船便利度简标

风险信息要按级别展示：

- 普通封禁、动态管控、作业区边界、临时限制，这类信息不从推荐列表中移除，统一放入第三层级“风险与注意事项”
- 只有明确存在生命危险型威胁时，才允许在第二层级卡片前置醒目危险提示
- 生命危险型威胁包括：极度湿滑的消波块、无护栏临深水边缘、涨潮后可能被困、强风浪导致落水风险高、夜间失足后自救困难

### 第二层级必要 UI

为了 App 化体验，我建议这个页面额外具备：

- 顶部返回按钮
- 当前模式标题
- 搜索按钮
- 筛选按钮
- 排序说明入口
- 固定在顶部的轻量筛选栏
- 二级危险提示徽标位

### 第二层级建议筛选器

海岸钓页建议：

- 区域
- 目标鱼
- 结构类型
- 新手友好度
- 是否适合夜钓

出海钓页建议：

- 区域
- 目标鱼
- 海域类型
- 成本等级
- 新手友好度

注意：

- 筛选器要轻，不要做成复杂后台
- 默认仍然保持“按推荐排名排序”

## 第三层级：单钓点详情页

第二层级点击“展开详细”后，跳转到第三层级。

第三层级是单独页面，不停留在第二层级内部展开，也不保留第二层级的大量钓点卡 UI。

### 第三层级页面目标

- 让用户完整理解这个钓点值不值得去
- 让用户知道什么时候去、怎么钓、注意什么
- 让用户从当前点继续跳到相近或更强的相关推荐

### 第三层级内容结构

#### 1. 顶部信息

- 返回按钮
- 钓点名称
- 模式标签：海岸钓 / 出海钓
- 排名标签

#### 2. 主图区域

- 钓点主图
- 图片说明

#### 3. 基本信息区

这一块承接第二层级的基础信息：

- 区域与导航信息
- 核心钓段说明
- 主打鱼种
- 核心汛期

#### 4. 实战玩法区

- 推荐钓法与线组
- 推荐饵料
- 黄金窗口期
- 适合谁去

#### 5. 评估区

- 新手友好度
- 风险与注意事项
- 合规提醒

#### 6. 推荐区

- 相关推荐
- 同区域更强点
- 同目标鱼推荐点

#### 7. 信息区

- 信息来源
- 更新时间
- 待复核提示

### 第三层级必要 UI

- 返回上一页
- 收藏按钮
- 分享按钮
- 查看下一个推荐点按钮

第三层级不建议保留：

- 第二层级整屏钓点列表
- 第二层级筛选条
- 首页双模式大入口

## 风险信息展示原则

### 1. 不因为存在管控信息就删除推荐钓点

如果一个点位本身很强，但存在：

- 港区动态管控
- 工业区可进入性不稳定
- 停车限制
- 作业期临时限制

这些内容不影响它作为“好钓点”被保留，只写入第三层级“风险与注意事项”。

### 2. 第二层级只前置生命危险型提示

第二层级是浏览和决策页，只有会直接影响人身安全的威胁才前置提示，例如：

- 消波块极滑
- 深水无护栏
- 夜钓坠落风险高
- 涨潮后易被困
- 强浪冲击明显

### 3. 第三层级承接全部风险与限制

第三层级必须完整承接：

- 安全风险
- 动态管控
- 合规提醒
- 进入性限制
- 不建议谁去

## 视觉与设备原则

### 1. 以 iPhone 竖屏为第一设计基准

设计基准建议以苹果常见竖屏逻辑宽度来做：

- 390pt
- 393pt
- 402pt

这对应近年的主流 iPhone 宽度区间。

因此页面设计要优先保证：

- 单手拇指操作舒服
- 顶部安全区处理正确
- 底部 Home Indicator 安全区处理正确

### 2. 视觉层次

- 第一层级：像 App 首页入口
- 第二层级：像推荐列表
- 第三层级：像单卡详情页

### 3. 内容优先级

全站优先级统一如下：

1. 钓点排名
2. 钓点基本信息
3. 玩法与风险
4. 辅助内容

## 图片策略

因为这是钓点导向网站，图片不能只是装饰。

第二层级和第三层级都建议配图，但不同层级用法不同：

- 第二层级：用于快速识别点位环境
- 第三层级：用于帮助理解钓点地貌、结构、环境气质

图片来源建议分级处理：

1. 官方景区/官方宣传图
2. 官方报道图片
3. 能明确对应地点的公开图片
4. 若高质量公开图不足，则先用区域环境图或结构示意图占位

对工业港区、防波堤、港池类点位要特别注意：

- 公开高质量图片不一定齐全
- 可优先使用环境图、远景图、码头图、示意图
- 后续如有实拍素材，再替换为更精准的点位图

## 现在可以直接进入制作的页面顺序

如果按你的思路推进，我建议开发顺序就是：

1. 第一层级：首页
2. 第二层级：海岸钓列表页
3. 第二层级：出海钓列表页
4. 第三层级：单钓点详情页

做完这 4 个核心页面，网站主干就成立了。
