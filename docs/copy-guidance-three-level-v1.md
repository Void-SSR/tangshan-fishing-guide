# 指导型网站三层级逐字稿事实边界 v1

## A. 首页能说什么，不能说什么

### 能说什么

- 这是一个`按出钓方式进入`的指导型网站。
- 首页只负责帮助用户先做第一步决策：`海岸钓`还是`出海钓`。
- 网站按`钓点推荐强度`组织，不是按资讯流组织。
- 推荐逻辑可以明确说成：优先推荐`鱼更多、稳定性更高、实战价值更强`的钓点。
- 可以说当前信息覆盖的核心区域和模式，例如曹妃甸、乐亭等，但只作轻量说明。
- 可以放`排名说明`、`合规说明`、`最近更新`入口。

### 不能说什么

- 不能在首页直接展开大量钓点详情。
- 不能在首页先讲完整玩法、线组、风险细节，这些属于第三层级。
- 不能把首页写成“新闻首页”或“综合门户”。
- 不能把内部强判断原词直接外放，例如“爆护天花板”“断层 TOP 级”，应改成稳健表达，如“高产出核心点位”“优先推荐点位”。
- 不能把动态信息写成绝对结论，例如港区进入性、休渔限制、临时管控。
- 不能在首页混写海岸钓和出海钓的具体推荐榜单。

## B. 第二层级每张卡必须承载的事实字段及顺序

### 每张卡必须有的字段

1. 钓点图片
2. 钓点名称
3. 模式标签：海岸钓 / 出海钓
4. 排名标识：第几推荐 / 所属梯队
5. 区域与导航信息
6. 核心钓段说明
7. 主打鱼种
8. 核心汛期
9. 查看详细按钮

### 推荐顺序说明

- 先让用户一眼识别`这是什么点`：图片、名称、模式、排名。
- 再让用户快速判断`值不值得去`：区域、核心钓段、主打鱼种、汛期。
- 最后给动作：`查看详细`。

### 第二层级可补但不能压过主信息的字段

- 新手友好度简标
- 风险简标
- 停车便利度 / 登船便利度简标

### 第二层级风险前置边界

- 只前置`生命危险型威胁`
- 例如：深水无护栏、消波块极滑、涨潮易被困、夜间失足后自救困难
- 普通管控、作业区边界、港区进入性、临时限制，不前置到第二层级

## C. 第三层级哪些内容必须承接，哪些内容不该前置到上一层

### 第三层级必须承接的内容

- 第二层级的全部基础信息：
  - 钓点名称
  - 模式标签
  - 排名标签
  - 区域与导航信息
  - 核心钓段说明
  - 主打鱼种
  - 核心汛期
- 实战信息：
  - 推荐钓法与线组
  - 推荐饵料
  - 黄金窗口期
  - 适合谁去
- 评估信息：
  - 新手友好度
  - 风险与注意事项
  - 合规提醒
- 决策补充：
  - 相关推荐
  - 信息来源
  - 更新时间
  - 待复核提示

### 不该前置到第一层级的内容

- 单点玩法
- 单点线组和饵料
- 单点风险细节
- 相关推荐
- 具体导航细节

### 不该前置到第二层级的内容

- 完整钓法说明
- 完整饵料说明
- 普通管控与进入限制全文
- 合规细则全文
- 信息来源与更新时间全文
- 同区域 / 同目标鱼完整推荐逻辑

### 第三层级表达底线

- 动态信息必须保留“待复核”意识，不写死为永久事实。
- 普通限制不删点，只放在第三层级风险区。
- 第三层级的目标不是继续做列表，而是让用户完成“去不去、何时去、怎么去、怎么钓”的决策。
