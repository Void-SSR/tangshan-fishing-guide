# 唐山钓鱼指南 UI 方向 v1

## 角色说明

本文档只负责 UI 与前端体验方向，建立在现有三层 App 型架构之上：

1. 第一层级：首页
2. 第二层级：钓点列表页
3. 第三层级：单钓点详情页

本产品的核心不是做资讯站，而是做一个围绕钓点排名与出行决策的移动端工具。

---

## 总体设计主张

### 视觉气质

- 关键词：冷静、专业、海岸感、决策感
- 视觉基调：深海蓝、雾灰、沙石灰，配少量锈橙作为强调色
- 页面感受：像一个可信的户外工具，而不是一个热闹的信息门户

### 信息表达原则

- 先告诉用户哪里最值得去，再告诉用户怎么去、怎么钓
- 所有页面都优先服务决策，不让用户在视觉噪声里找重点
- 排名、模式、风险、行动按钮必须第一眼可见

### App 化原则

- 全站只按 iPhone 竖屏优先设计
- 所有关键操作都在单手可达区域内
- 页面之间是明确跳转，不做网页式堆叠长流
- 每层页面只承担自己的任务，不串层复用大块 UI

---

## 设备与尺寸基准

### iPhone 竖屏基准

- 设计基准宽度：390pt
- 兼容宽度区间：375pt - 430pt
- 优先适配：iPhone 13 / 14 / 15 / 16 常见竖屏尺寸

### 安全区规则

- 顶部内容必须避开刘海与动态岛，顶部主标题区应预留安全区外加 12-16pt 缓冲
- 底部固定操作区必须避开 Home Indicator，底部操作栏内部留白不得少于 12pt
- 页面首屏主操作按钮和底部操作按钮都不能贴边

### 基础尺寸建议

- 页面左右边距：16pt
- 模块间距：16-24pt
- 卡片圆角：18-24pt
- 主按钮高度：52-56pt
- 次按钮高度：44-48pt
- 点击热区最小：44pt x 44pt
- 标签高度：24-28pt

## 第一层级：首页

### 视觉目标

- 第一眼就让用户明白这是“唐山海钓指南”
- 第二眼就看懂只有两个选择：海岸钓、出海钓
- 视觉上要稳、要强，不像资讯首页，也不像活动海报

### 信息节奏

首页的信息节奏必须极短，按三拍完成：

1. 品牌与一句话说明
2. 两张模式入口大卡
3. 极少量辅助信息

首页绝不承载大量钓点内容，不抢二级页的工作。

### 交互重点

- 用户进入首页后，最重要的动作就是二选一
- 两张模式卡都必须足够大，点击无歧义
- 点击后直接进入对应模式列表页，不在首页做中间浮层或复杂选择

### 页面结构建议

#### 1. 顶部品牌区

- 标题：唐山钓鱼指南
- 副标题：优先推荐更强、更稳、鱼更多的海岸钓与出海钓钓点

#### 2. 模式入口区

- 海岸钓入口卡
- 出海钓入口卡

每张卡建议包含：

- 背景图或环境图
- 模式名
- 一句短说明
- 明确的进入按钮

#### 3. 轻辅助区

- 排名说明入口
- 最近更新时间
- 合规提示入口

### 首页视觉建议

- 主图不需要很复杂，但要有海风、堤岸、船只或礁区的环境暗示
- 两张模式卡风格一致，避免一张像广告、一张像系统按钮
- 主按钮文案短，优先使用“进入海岸钓”“进入出海钓”

## 第二层级：钓点列表页

### 页面定位

这是浏览效率页，不是内容阅读页。

用户进入这个页面的目的，是快速比较当前模式下大量钓点，按推荐强度找到最值得点开的目标。

### 视觉目标

- 整体要像一个高效率的移动端推荐列表
- 钓点卡既要有吸引力，也要高度规整
- 排名、钓点名称、主打鱼种和核心汛期必须扫一眼就能抓到

### 信息节奏

第二层级的信息节奏应当是：

1. 模式确认
2. 筛选与搜索
3. 高密度钓点卡浏览
4. 单点跳转详情

用户在这里是“滑动决策”，不是“沉浸阅读”。

### 交互重点

- 返回按钮明确
- 搜索和筛选要轻，但够用
- 默认按推荐强度排序，不让用户一进来就迷路
- 卡片整体可点，底部再给一个显式“查看详细”按钮增强确定感

### 顶部结构建议

- 左侧：返回
- 中间：当前模式标题，如“海岸钓”
- 右侧：搜索、筛选

标题下建议放一条简短说明：

- 默认按推荐强度排序

### 筛选器建议

筛选器使用横向滑动 Chip，不做复杂下拉堆叠。

海岸钓建议：

- 区域
- 目标鱼
- 结构类型
- 新手友好
- 夜钓

出海钓建议：

- 区域
- 目标鱼
- 海域类型
- 成本等级
- 新手友好

### 列表卡片规范

#### 图片比例

- 推荐比例：4:3
- 卡片顶部大图，保持稳定统一
- 图片必须作为“识别钓场环境”的第一线索，不只是装饰

#### 卡片内容顺序

1. 图片
2. 角标层：排名标签、模式标签、危险提示
3. 钓点名称
4. 区域与导航信息
5. 核心钓段说明
6. 主打鱼种
7. 核心汛期
8. 查看详细按钮

#### 卡片布局原则

- 重要信息尽量控制在 2-4 行一个块
- 避免一张卡里塞太多表格化数据
- 核心钓段说明最多 2 行，超出用省略

### 排名标签

- 排名标签必须始终在图片层上方，用户无需下滑就能看到
- 第 1 推荐、第 2 推荐这类文案比单纯数字更直接
- 第一梯队可使用更强视觉对比，但不要做成夸张促销贴纸

### 危险提示

第二层级只显示生命危险型威胁。

允许出现在卡片顶部角标或图片下方首行：

- 深水无护栏
- 消波块极滑
- 涨潮易被困
- 夜间坠落风险高

不在第二层级前置：

- 动态管控
- 港区封禁
- 停车限制
- 作业区限制

这些统一放第三层级的“风险与注意事项”。

### 列表浏览效率细节

- 每屏建议完整展示 1.3-1.6 张卡，形成继续下滑动力
- 卡片之间要有明显分段，不做资讯流无边界拼接
- 搜索和筛选栏可吸顶，但高度要轻，不能压缩卡片浏览空间

### 第二层级底部策略

- 不设置固定底部主操作栏，避免挤压列表浏览空间
- 可在滚动一定距离后出现“返回顶部”悬浮按钮

## 第三层级：单钓点详情页

### 页面定位

这是单点沉浸页。

用户从第二层级进入第三层级后，目标不是再比较大量点位，而是完整理解这个点值不值得去、该怎么去、该怎么钓、要注意什么。

### 视觉目标

- 形成明显的“进入单钓点世界”的感觉
- 页面要更沉浸、更稳定、更可信
- 重点信息要像作战卡，而不是像一篇长文章

### 信息节奏

第三层级建议按照“判断 -> 准备 -> 风险 -> 延展”的顺序展开：

1. 判断这个点值不值得去
2. 了解什么时候去、怎么钓
3. 明确风险和限制
4. 再看相关推荐

### 交互重点

- 返回上一页必须明确
- 页面顶部就要给出排名和一句话判断
- 底部操作要帮助用户继续前进，而不是把用户扔回列表里迷路

### 页面结构建议

#### 1. 顶部导航区

- 返回按钮
- 模式标签
- 排名标签
- 收藏按钮
- 分享按钮

#### 2. 主图区

- 16:9 或 4:3 主图均可，但详情页更推荐 16:9，沉浸感更强
- 主图下用一句图注告诉用户该看哪里

#### 3. 核心判断区

- 钓点名称
- 一句话判断
- 区域与导航
- 当前点位适合谁去

#### 4. 基本信息区

- 核心钓段说明
- 主打鱼种
- 核心汛期
- 黄金窗口期

#### 5. 实战玩法区

- 推荐钓法与线组
- 推荐饵料
- 新手友好度

这一块要做成清晰分段，不做长段文字堆积。

#### 6. 风险与注意事项区

- 危险先写前面
- 管控和限制写后面
- 语言直接，不绕弯

#### 7. 相关推荐区

- 同区域推荐
- 同目标鱼推荐
- 下一个推荐点

### 第三层级沉浸与清晰细节

- 一级标题和钓点名称字号要明显大于正文
- 每个信息模块都像单独卡片，便于跳读
- 不允许一个区块同时承担三种以上信息任务
- 相关推荐要在正文后部，不打断主阅读流

### 第三层级底部操作区

建议设置固定底部操作区，适配单手操作：

- 左侧：返回列表
- 右侧：查看下一个推荐点

如果空间足够，也可加入“收藏”图标，但不能抢主按钮。

## 组件规范

### 按钮

- 主按钮：深色底，高对比白字，圆角 16-18pt
- 次按钮：浅底描边或浅色实底
- 危险提示按钮或标签：红橙色系，但控制面积，只用于真正危险

### 标签

- 模式标签：海岸钓 / 出海钓
- 排名标签：第 1 推荐 / 第一梯队
- 状态标签：新手友好 / 夜钓优先 / 风险高

标签必须短，不允许做成长句按钮。

### 图片

- 列表页图片优先展示环境识别度
- 详情页图片优先展示空间关系和结构重点
- 若没有现场图，可退而使用：
  - 地图标注图
  - 航拍位置图
  - 博主公开定位图
  - 海岸结构示意图

### 字体层级建议

- 页面主标题：28-32pt
- 钓点名称：24-28pt
- 模块标题：18-20pt
- 正文：15-16pt
- 标签与说明：12-13pt

正文行距建议保持在 1.45-1.6 区间，保证移动端可读性。

## 设计红线

1. 不能做成资讯门户首页。首页不能堆新闻流、文章流或大量混杂卡片。
2. 不能把首页双入口带到第二层级。进入海岸钓或出海钓后，就只服务当前模式。
3. 不能把第二层级做成长文章列表。第二层级必须以高效率卡片浏览为核心。
4. 不能把第三层级做成二级页原地展开。第三层级必须是独立单点详情页。
5. 不能把危险提示滥用到所有卡片。第二层级只前置生命危险型威胁。
6. 不能为了美观牺牲排名和鱼情信息。排名、主打鱼种、核心汛期必须始终优先于装饰。
7. 不能做成桌面网页思路。所有布局、按钮、间距都要以 iPhone 单手使用为前提。
8. 不能使用过多花哨动效。动效只用于层级切换和操作反馈，不做炫技。

---

## 三层页面设计主张总结

### 首页

首页的主张是“少而准”，让用户在 3 秒内完成模式选择。

### 第二层级

第二层级的主张是“快而清楚”，让用户高效率浏览大量钓点卡，迅速做出点击决定。

### 第三层级

第三层级的主张是“沉浸但不混乱”，把单个钓点讲透，让用户带着明确准备离开页面。
