# 唐山海钓指南 QA 与 UX 验收清单 v1

## 文档定位

这份清单用于验收“唐山海钓指南”App 型网站的三层结构、导航行为、风险表达、iPhone 适配和前端稳定性。

它不是视觉稿，也不是产品需求原文，而是开发、设计、测试三方共用的验收标准。

## 核心验收红线

以下 5 条属于 P0 级红线，任意一条不满足，都不能算通过验收：

1. 第一层级首页只能承载模式选择，核心入口只有“海岸钓”和“出海钓”。
2. 用户从第一层级进入第二层级后，页面中不能再出现第一层级的双入口大按钮 UI。
3. 用户从第二层级进入第三层级后，页面中不能再保留第二层级的大列表 UI、整屏卡片流或筛选条。
4. 第三层级必须是单一钓点的独立详情页，不能是二级列表页中的折叠展开状态。
5. 第二层级只允许前置“生命危险型威胁”，普通管控、动态封禁、进入限制等内容只能放在第三层级“风险与注意事项”。

## 验收范围

验收重点覆盖：

- 第一层级：首页
- 第二层级：海岸钓列表页 / 出海钓列表页
- 第三层级：单钓点详情页
- 三层之间的跳转与返回
- iPhone 竖屏适配
- 危险提示与风险表达
- 筛选、排序、搜索、状态保持

## 第一层级验收清单

### 信息结构

- 页面标题与产品名称清晰可见，用户能在 3 秒内理解这是唐山海钓指南。
- 首页只呈现模式选择，不直接堆叠大量钓点卡。
- 首页只有两个核心入口：`海岸钓`、`出海钓`。
- 首页可以有轻量说明、更新时间、排名说明、合规提醒入口，但不能压过模式入口。
- 首页不混排海岸钓和出海钓钓点列表。

### 交互行为

- 点击 `海岸钓` 必须跳转到海岸钓第二层级列表页。
- 点击 `出海钓` 必须跳转到出海钓第二层级列表页。
- 首页不应出现“展开详细”类入口，因为首页不承担单点浏览职责。
- 从首页进入第二层级后，浏览器返回应回到首页，而不是空白页或错误页。

### UI 与移动端体验

- 首页首屏内能完整看到两个模式入口的核心信息，不出现被安全区遮挡的情况。
- 顶部内容不应压进 iPhone 刘海或状态栏区域。
- 底部按钮、说明或更新时间不应贴住 Home Indicator。
- 两个模式入口的点按区域足够大，单手操作不易误触。
- 首屏不应出现横向滚动。

### 首页常见失败判定

- 首屏出现多个钓点卡，导致首页变成列表页。
- 海岸钓和出海钓入口视觉权重差异过大，用户误以为只有一个可选项。
- 点击入口后仍然停留在首页，通过局部展开显示二级内容。
- 进入二级后，通过悬浮层仍能看到首页双入口大卡。

## 第二层级验收清单

### 页面定位

- 海岸钓列表页只展示海岸钓点。
- 出海钓列表页只展示出海钓点。
- 当前页面模式必须明确可见，例如页面标题、模式标签、顶部栏。
- 页面默认按推荐强度排序，优先展示鱼更多、表现更强的点位。

### 必备信息字段

每张钓点卡都必须包含：

- 钓点图片
- 钓点名称
- 船钓/岸钓标签
- 区域与导航信息
- 核心钓段说明
- 主打鱼种
- 核心汛期
- 排名标识
- 进入第三层级的按钮，例如“查看详细”或“展开详细”

### 结构与跳转

- 从首页进入第二层级后，不再出现首页的“海岸钓 / 出海钓”双入口大按钮。
- 第二层级可以有返回按钮，但返回按钮的作用是回到首页，而不是再次渲染首页内容到当前页面中。
- 点击任意钓点卡的详细按钮，必须跳转到对应钓点的第三层级独立详情页。
- 第二层级不允许在当前列表页内直接展开完整玩法、长段风险说明、相关推荐整块内容。
- 第二层级不允许通过手风琴、抽屉、弹层来替代第三层级详情页。

### 排序与筛选

- 默认排序必须符合当前信息库中的钓点推荐顺序。
- 筛选后应保留“当前模式内排序”的逻辑，不能把海岸钓和出海钓混在一起。
- 清空筛选后，应恢复默认推荐排序。
- 搜索、筛选、排序切换后，不应出现重复卡片、空白占位卡片或顺序错乱。
- 从第三层级返回第二层级时，应保留用户上一次的滚动位置、筛选条件和搜索词。

### 危险提示规则

- 只有生命危险型威胁，才能在第二层级卡片上前置危险提示。
- 危险提示应采用统一视觉样式，不应和普通标签混淆。
- 危险提示必须简短明确，例如：`生命危险：涨潮易被困`、`生命危险：消波块极滑`。
- 没有生命危险型威胁的钓点，不允许滥用红色高危提示。
- 港区动态管控、临时封禁、停车限制、作业时段限制，不应出现在第二层级前置危险提示位。

### 第二层级 UI 与移动端体验

- 顶部栏可见当前模式名与返回动作。
- 筛选入口清晰，但不应遮挡列表首卡内容。
- 顶部筛选条若吸顶，滚动时不应抖动、遮挡标题或吞掉首卡。
- 钓点卡图片比例统一，不应因为图片尺寸不同导致卡片高度异常跳动。
- 卡片主要点击目标明确，避免整卡和按钮点击行为冲突。
- 长文案在 iPhone 窄屏下不会溢出容器。

### 第二层级常见失败判定

- 海岸钓页里混入出海钓点，或反之。
- 从首页进入二级后，顶部或底部仍出现首页双入口按钮。
- 点击“查看详细”后在当前列表页下方展开详情，而不是跳转新页面。
- 危险提示把“封禁、管控、停车难”错误当成生命危险前置。
- 返回列表后筛选状态丢失，用户需要重新筛选。

## 第三层级验收清单

### 页面定位

- 第三层级必须是一页只讲一个钓点。
- 用户进入第三层级后，页面结构必须清楚表明这是“单钓点详情页”。
- 页面标题、主信息、图片与内容都必须与当前钓点唯一对应，不能串点。

### 必备内容

第三层级必须承接第二层级基础信息，并补全以下内容：

- 钓点主图
- 区域与导航信息
- 核心钓段说明
- 主打鱼种
- 核心汛期
- 推荐钓法与线组
- 推荐饵料
- 新手友好度
- 风险与注意事项
- 相关推荐
- 信息来源
- 更新时间

### 结构与导航

- 第三层级不能保留第二层级整屏钓点列表。
- 第三层级不能保留第二层级吸顶筛选条。
- 第三层级不能再次出现首页双模式大入口。
- 返回按钮应回到用户离开的第二层级，并尽量恢复原滚动位置和筛选状态。
- “查看下一个推荐点”应跳转到下一个独立详情页，而不是把下一个点追加在当前页下方。
- 分享、收藏等操作不应干扰主阅读路径。

### 风险表达

- 第三层级承接全部风险与限制信息。
- 普通管控、动态封禁、港区边界、作业时段、停车限制、补给缺失等，应写入“风险与注意事项”。
- 生命危险型威胁在第三层级仍需完整解释，不应只在第二层级给一个标签后就消失。
- 风险文案必须区分“人身安全风险”和“进入性/规则风险”，避免误导用户。
- 不能把“建议复核”写成“禁止前往”，也不能把“危险极高”写得像普通提醒。

### 第三层级 UI 与移动端体验

- 主图加载失败时有占位，不应让页面顶部塌陷。
- 长内容分区清楚，支持连续阅读，不出现一屏巨长密文。
- 页面底部操作按钮不会遮挡最后一段“风险与注意事项”或“信息来源”。
- iPhone 竖屏下，正文文字、标签、按钮、图片说明都保持可读性。
- 内容区不应出现横向滚动。

### 第三层级常见失败判定

- 详情页顶部还能看到二级列表中的多个钓点卡缩略区。
- 路由参数错误，打开 A 点却显示 B 点内容。
- 返回后直接回到首页，导致用户丢失原列表上下文。
- 同一个详情页里同时出现海岸钓和出海钓混合相关推荐。
- 风险说明过短，无法区分“生命危险”与“动态管理”。

## 生命危险型威胁定义

只有满足“对人身安全有直接、明确、现实威胁”的情况，才属于第二层级需要前置提示的生命危险型威胁。

### 需要在第二层级前置提示的情况

- 消波块、礁石、坝体极度湿滑，失足后直接有坠海或重伤风险。
- 深水边缘无护栏，且站位空间狭窄，存在明显坠落风险。
- 涨潮后退路可能消失，存在被困、落水或无法撤离风险。
- 风浪强、浪涌高、拍岸冲击明显，靠边站位存在被卷落风险。
- 夜钓环境极差、照明不足、救援困难，失足后自救概率低。
- 土路塌陷、边坡松动、脚下结构不稳，可能直接导致跌落或被困。

### 不应在第二层级前置危险提示的情况

- 港区动态管控
- 临时封闭
- 进入权限不稳定
- 停车不便
- 饵料补给不足
- 作业车辆较多
- 建议结伴
- 雨雪天不建议轿车前往

以上内容可写进第三层级“风险与注意事项”，但不属于二级前置的生命危险提示。

## 最容易出现的前端 Bug 与 UX 问题

### 导航与路由问题

- 首页进入二级后，通过组件复用把首页双入口卡继续渲染在二级页。
- 二级进入三级时采用局部展开而不是页面跳转，破坏三层结构。
- 三级返回时没有恢复二级列表的滚动位置。
- 三级返回时直接回首页，丢失用户上下文。
- 深链接进入三级后，返回行为异常，出现死循环或空白页。

### 列表与筛选问题

- 筛选后排序失效，第一梯队被排到后面。
- 海岸钓与出海钓共用同一筛选状态，导致跨模式污染。
- 搜索词、筛选条件、排序说明在切换页面后被错误重置。
- 快速切换筛选时，卡片重复、闪烁或残留旧数据。
- 二级列表分页或懒加载后，排名编号错乱。

### 详情页内容问题

- 当前钓点详情与相关推荐数据串线。
- 钓点图片、导航信息、主打鱼种、核心汛期来自不同点位，形成拼接错误。
- 风险与注意事项被隐藏在折叠区，用户默认看不到。
- 信息来源和更新时间缺失，影响可信度判断。

### 安全区与移动端布局问题

- 顶部标题压进刘海区。
- 底部固定按钮盖住最后一段正文。
- 卡片宽度或标签过长，导致 iPhone 窄屏横向滚动。
- 吸顶筛选条在 Safari 中抖动，滚动时覆盖首张卡片。
- 图片比例不统一，列表滚动时发生明显布局跳动。

### 风险表达问题

- 把“动态管控”写成“危险”，误导用户。
- 把“生命危险”埋在长段落里，没有前置表达。
- 同一页面同时使用“危险”“封禁”“限制”“不建议”但没有层级，用户难以判断轻重。
- 为了显得严肃，把过多钓点都标成高危，破坏推荐逻辑与阅读效率。

## 建议测试路径

### 路径 1：首页到海岸钓

1. 打开首页。
2. 点击 `海岸钓`。
3. 确认进入海岸钓第二层级。
4. 确认页面中不再出现首页双入口大按钮。

### 路径 2：首页到出海钓

1. 打开首页。
2. 点击 `出海钓`。
3. 确认进入出海钓第二层级。
4. 确认页面中不再出现首页双入口大按钮。

### 路径 3：第二层级到第三层级

1. 在第二层级点击某个钓点的 `查看详细`。
2. 确认进入独立详情页。
3. 确认页面中不保留第二层级大列表 UI。
4. 确认风险、玩法、相关推荐、来源和更新时间完整可见。

### 路径 4：第三层级返回

1. 先在第二层级滚动到中后段。
2. 设置一个筛选条件。
3. 进入第三层级。
4. 点击返回。
5. 确认回到原列表位置，且筛选条件保留。

### 路径 5：危险提示校验

1. 选择一个存在生命危险型威胁的点位。
2. 确认第二层级有前置危险提示。
3. 进入第三层级，确认该危险有完整解释。
4. 再选择一个仅有动态管控的点位。
5. 确认第二层级无高危提示，但第三层级有风险说明。

## 建议验收设备

- iPhone 12 / 13 / 14 标准尺寸宽度
- iPhone 14 Pro / 15 Pro 动态岛尺寸
- iPhone Plus / Pro Max 大屏尺寸
- Safari 移动端环境优先

## 最高优先级风险总结

- 三层结构被实现成“同页展开”而不是“页面跳转”，这是当前最容易偏离需求的风险。
- 二级页复用首页模块，导致首页双入口在二级继续出现，这是最容易破坏结构纯度的问题。
- 三级页如果没有承接返回状态，用户回列表后丢失筛选和滚动位置，体验会非常差。
- 风险提示如果不区分“生命危险”和“动态管控”，会同时伤害可信度和推荐逻辑。

## 关键验收项总结

- 一级进二级后，不再出现一级按钮。
- 二级进三级后，不再出现二级大列表 UI。
- 三级必须是独立单钓点详情页。
- 第二层级只前置生命危险型威胁。
- 第三层级承接全部风险、玩法、相关推荐、来源和更新时间。

## 最容易踩坑的问题

- 用弹层、抽屉或手风琴偷换第三层级。
- 返回逻辑没做状态恢复。
- 危险标签滥用，导致二级页“全红”。
- iPhone 底部安全区处理不好，按钮压住正文。
- 同一个详情页拼错数据，出现图片、鱼种、导航信息串点。
